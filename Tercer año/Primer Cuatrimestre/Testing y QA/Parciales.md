# 1er Parcial

### 1. Que diferencias hay entre verficacion y validacion?
- **Verificación:** consiste en evaluar si el software se está construyendo correctamente según las especificaciones definidas. Responde a la pregunta: _“¿Estoy construyendo correctamente el producto?”_.
- **Validación:** consiste en evaluar si el software desarrollado cumple con las necesidades y expectativas del usuario. Responde a la pregunta: _“¿Estoy construyendo el producto correcto?”_.

### 2. Que es error - fallas - defecto - incidentes? 
- **Error:** es una acción humana incorrecta que produce un problema en el software (por ejemplo, un error de programación o de diseño).

- **Defecto (bug):** es la consecuencia del error, es decir, una falla en el código o en un componente del sistema.

- **Falla:** es la manifestación del defecto cuando el software se ejecuta, es decir, cuando el sistema no se comporta como se esperaba.

- **Incidentes:** Eventos que ocurren al ejecutar casos de prueba que requieren análisis para determinar si son fallas.

### 3. Que es caja negra? 
**Testing de caja negra:** Es una técnica de prueba en la que se evalúa el comportamiento del sistema sin conocer su estructura interna ni el código fuente.  
Se basa en verificar las entradas y salidas del sistema, comprobando que cumpla con los requisitos funcionales.

### 4. Que es Testing? 
Es el proceso de operar un sistema o componente en condiciones determinadas, observando y registrando los resultados para una evaluacion, con el objetivo de detectar defectos y evaluar la calidad del software.

### ==5. Cual es la guia para analisis de frontera?== 
El **análisis de frontera** es una técnica de testing que consiste en probar los valores límite de un dominio de entrada, ya que es donde suelen encontrarse más errores.

### ==6. Defina la diferencia entre condicion de prueba y casos de prueba.==
- **Condición de prueba:** es un aspecto, requisito o situación del sistema que se desea verificar (qué se quiere probar).

- **Caso de prueba:** es la definición concreta de cómo se va a probar esa condición, incluyendo entradas, pasos, resultados esperados y resultados obtenidos.

### 7. ¿Qué diferencias hay entre Control de calidad vs  Aseguramente de calidad?
**Aseguramiento de la Calidad (QA):** es el conjunto de actividades orientadas a asegurar que se cumplan los procesos, estándares y planes definidos. Su foco está en el **proceso**.
**Control de Calidad (QC):** es la evaluación del producto para detectar defectos o problemas. Su foco está en el **producto de software**.
